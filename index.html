Veloighter

Velog라는 블로그 사이트에서 원하는 사용자의 글을 팔로잉 할 수 있도록 도와주는 프로그램입니다.

Details

  • Tags Side Project
  • Role Front-end Dev React Javascript TypeScritpt
  • 팀원 수 2 (Front-end Dev & Back-end Dev)
  • 소요 시간 1 개월

Motivation

Velog는 개발자들이 글을 쓰고 공유하는 사이트입니다. Velog에서 구독하고 싶은 사용자의 글을 볼 수 있는 카테고리가 없었습니다. 구독하고 싶은 사용자의 글을 표시만 해줘도 조금이라도 사용자들이 Velog를 사용하는데 도움이 되지 않을까라는 생각을 했습니다. 그 생각을 구현시키기 위해 크롬 확장 프로그램을 이용해 Veloighter라는 프로그램을 만들게 되었습니다.

Development

  • Popup

    팝업 파일은 사용자가 툴바 아이콘을 눌렀을 때 나오는 창에 표시되는 HTML 파일입니다. 스타일 시트를 링크하는 태그와 스크립트 태그를 포함하는 면에서 웹 페이지와 비슷하게 동작합니다. 하지만 웹 페이지와 달리 인라인 자바스크립트는 허용하지 않습니다.

    React와 TypeScritpt를 사용하여 팝업 파일을 작성하였습니다. 스토리지에 저장 되어있는 사용자 정보와 팝업 화면에 나오는 정보가 동일하게 나와야 했습니다. 그래서 팝업 파일을 작성시 가장 중요하게 생각했던 부분은 스토리지에 저장되어 있는 사용자 정보와 팝업 페이지의 클래스의 state에 있는 정보를 동일화 시키는 것이었습니다.

    전체 코드 보기 자세한 설명글 보기
  • Content Script

    콘텐츠 스크립트는 익스텐션의 기능이 적용될 웹페이지에서 작동하는 파일입니다. 콘텐츠 스크립트는 DOM을 사용하여 브라우저가 방문한 페이지를 바꿀 수 있습니다.

    Velog의 피드는 무한대로 로드되기 때문에 피드에 있는 사용자의 이름을 실시간으로 가져와 스토리지에 저장 되어있는 사용자의 이름과 비교하는것에 초점을 맞췄습니다. 사용자의 이름을 실시간으로 가져오기 위해 getElementByClassName을 사용하였고, 팔로우한 사용자의 글을 표시해주거나 언팔로우한 사용자의 글의 표시된 부분을 제거하는데 setInterval 메서드를 사용하였습니다.

    전체 코드 보기 자세한 설명글 보기
  • Storage

    익스텐션 사용자가 추가한 정보를 storage에 저장되게 하였습니다. storage를 활용하기 위해서는 chrome.storage API를 사용해야 합니다. storage.sync를 사용하여 크롬과 데이터가 자동적으로 동기화가 될 수 있도록 하였습니다.
